# i18n Design — Prode Mundial 2026

**Date:** 2026-06-08  
**Status:** Approved

## Overview

Add Spanish/English internationalization to the app using a minimal custom approach: no new dependencies, cookie-based locale detection, and a thin React Context for client components. The architecture is extensible — adding a third language requires only a new JSON file and one line of config.

---

## Decisions

| Question | Decision |
|---|---|
| Locale selection | Auto-detect from `Accept-Language` header; fallback to `es`. User can override with a toggle in the navbar. |
| Locale persistence | Cookie `prode_locale` (1 year, `SameSite=Lax`, `Path=/`). |
| URL structure | No change — no locale prefix. `/grupos` stays `/grupos` in both languages. |
| Library | None — custom implementation. |
| Default locale | `es` |
| Supported locales | `es`, `en` (extensible) |

## Architecture

### 1. `src/proxy.ts` — Locale detection

Extend the existing auth proxy to also detect the locale and forward it as a request header. This runs before every request so all Server Components can read it.

Logic (no new dependencies):
1. Read `prode_locale` cookie from the request.
2. If absent or invalid, parse `Accept-Language` manually: take the first tag, strip quality values, extract the two-letter language code (e.g. `"en-US,en;q=0.5"` → `"en"`).
3. If that language code matches a supported locale (`es` or `en`), use it. Otherwise default to `es`.
4. Build a new `Headers` object from the existing request headers, set `x-locale` to the resolved locale, and return `NextResponse.next({ request: { headers: newHeaders } })`.

Integration with the existing `auth()` wrapper: the current `proxy.ts` uses `auth((req) => { ... })`. The locale detection runs inside that callback, before the auth redirects, so that `NextResponse.next()` with the enriched headers is returned on pass-through requests. Auth redirects (`NextResponse.redirect(...)`) don't need the locale header.

### 4. `src/components/locale-provider.tsx` — Client-side access

```tsx
// Server usage (root layout):
const locale = await getLocale()
const dict = await getDictionary(locale)
return <LocaleProvider locale={locale} dict={dict}>...</LocaleProvider>

// Client component usage:
const { locale, dict, t } = useDict()
// t is pre-bound: t("nav.grupos") === dict.nav.grupos (no interpolation needed for static strings)
```

`LocaleProvider` is a `'use client'` Context. It receives `locale` and `dict` as serializable props from the Server Component root layout and makes them available to all client components via `useDict()`.

### 5. `src/components/locale-switcher.tsx` — Toggle in navbar

A `'use client'` component rendered inside the existing `<Navbar>`.

```tsx
// Visual: two small buttons  [ES] [EN]  — active one highlighted
// On click: calls setLocaleAction(newLocale)
```

`setLocaleAction` is a Server Action in `src/app/actions.ts`:
```ts
export async function setLocaleAction(locale: Locale) {
  cookies().set('prode_locale', locale, { maxAge: 60 * 60 * 24 * 365, path: '/', sameSite: 'lax' })
  revalidatePath('/', 'layout')
}
```

The `revalidatePath` call triggers a full layout re-render with the new locale — no redirect, no URL change.

### 6. Root layout updates (`src/app/layout.tsx`)

```tsx
const locale = await getLocale()
const dict = await getDictionary(locale)
return (
  <html lang={locale}>
    <body>
      <LocaleProvider locale={locale} dict={dict}>
        {children}
      </LocaleProvider>
    </body>
  </html>
)
```

## Data flow

```
Browser request
  ↓ proxy.ts: reads cookie/Accept-Language → sets x-locale header
  ↓ layout.tsx: getLocale() + getDictionary() → LocaleProvider
  ↓ Server Components: receive dict as prop or call getDictionary() directly
  ↓ Client Components: useDict() from LocaleProvider context
```

## Testing strategy

Existing unit tests (`src/__tests__/`) don't reference UI strings — no changes needed.

Component tests that render client components (`submit-button.test.tsx`, `prediction-input.test.tsx`, `invite-copy-button.test.tsx`, `bottom-nav.test.tsx`) need a `MockDictProvider` wrapper:

```tsx
// src/test-utils/dict-provider.tsx
import { LocaleProvider } from '@/components/locale-provider'
import esDict from '@/messages/es.json'

export function MockDictProvider({ children }: { children: React.ReactNode }) {
  return <LocaleProvider locale="es" dict={esDict}>{children}</LocaleProvider>
}
```

Wrap affected tests: `render(<MockDictProvider><ComponentUnderTest /></MockDictProvider>)`.
